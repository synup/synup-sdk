Synup SDK

Build local presence automations in minutes — review auto-responders, listings monitors, reputation dashboards, and more.

Python SDK v0.4.0
What you can build
  • Auto-reply to reviews across all locations
  • Bulk onboard 500+ locations from CSV

  • Generate weekly reputation reports

Installation

bash 
pip install synup-sdk

Quickstart

Auto-reply to positive reviews in 10 lines. Set SYNUP_API_KEY in your environment and run:

auto_responder.py 
import synup

client = synup.Synup()  # reads SYNUP_API_KEY from env

# Auto-reply to all 4-5 star reviews that haven't been answered
results = client.workflows.auto_reply_to_reviews(
    16808,
    template="Thanks for the {rating}-star review!",
    min_rating=4,
)
print(f"Replied to {len(results)} reviews")

Configuration

python
import synup

# Minimal — reads SYNUP_API_KEY env var
client = synup.Synup()

# Explicit key
client = synup.Synup(api_key="your-api-key")

# With custom config
client = synup.Synup(
    api_key="your-api-key",
    timeout=300.0,       # seconds (default: 240)
    max_retries=3,       # auto-retry on 429/5xx (default: 2)
)

Parameters

ParameterTypeDescription
api_keystrYour Synup API key. Reads from SYNUP_API_KEY env var if not provided.
base_urlstrAPI base URL. Default: https://api.synup.com
timeoutfloatRequest timeout in seconds. Default: 240.0
max_retriesintAuto-retry count on 429/5xx. Default: 2

Workflows

Pre-built automations that combine multiple API calls into single, product-level operations. These are real features, not just API wrappers.

Auto-reply to reviews

Fetches recent reviews, filters by rating and response status, and posts replies using your template.

python
# Dry-run first to preview
preview = client.workflows.auto_reply_to_reviews(
    16808,
    template="Thanks for the {rating}-star review!",
    min_rating=4,
    only_unanswered=True,
    dry_run=True,
)

# Then send for real
results = client.workflows.auto_reply_to_reviews(
    16808,
    template="Thanks for the {rating}-star review!",
    min_rating=4,
)
print(f"Replied to {len(results)} reviews")
ParameterTypeDescription
location_idstr | intLocation to process.
templatestrReply text. Use {rating} as placeholder.
min_ratingintOnly reply to reviews with this rating or higher. Default: 4
only_unansweredboolSkip reviews that already have a response. Default: True
dry_runboolPreview without sending. Default: False

Locations

Create, list, search, update, and organize business locations.

python
# List with pagination
page = client.locations.list(first=10)
for loc in page:
    print(loc.name, loc.city)

# Auto-paginate all
for loc in client.locations.list(first=100).auto_paging_iter():
    print(loc.name)

# Retrieve single location
location = client.locations.retrieve(16808)

# Search, filter by IDs, store codes, folder, or tags
page = client.locations.search("cafe", first=20)
locs = client.locations.list_by_ids([16808, 16749])
locs = client.locations.list_by_store_codes(["STORE01"])
locs = client.locations.list_by_folder(folder_name="franchise")
page = client.locations.list_by_tags(["vip"], first=20)

# Create, update, archive
result = client.locations.create({"name": "Acme", "city": "NYC", ...})
client.locations.update({"id": 16808, "phone": "5559876543"})
client.locations.archive([16808])

Error handling

The SDK raises specific exceptions so you can handle each failure mode differently.

python
import synup

client = synup.Synup()

try:
    client.locations.list()
except synup.AuthenticationError:
    print("Invalid API key")
except synup.RateLimitError as e:
    print(f"Rate limited — retry after {e.retry_after}s")
except synup.NotFoundError:
    print("Resource not found")
except synup.ValidationError as e:
    print(f"Bad request: {e.response_body}")
except synup.APIConnectionError:
    print("Network error")
ExceptionStatusWhen
AuthenticationError401Invalid or missing API key
PermissionDeniedError403Insufficient permissions
NotFoundError404Resource not found
ValidationError400/422Invalid request parameters
RateLimitError429Too many requests. Has retry_after attribute.
InternalServerError5xxServer-side error. Safe to retry.
APIConnectionError—Network failure
