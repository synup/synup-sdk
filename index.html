auto_responder.py 
import synup

client = synup.Synup()  # reads SYNUP_API_KEY from env

# Auto-reply to all 4-5 star reviews in one call
results = client.workflows.auto_reply_to_reviews(
    16808,
    template="Thanks for the {rating}-star review!",
    min_rating=4,
)
print(f"Replied to {len(results)} reviews")

# Full listings health audit
audit = client.workflows.listings_health_audit(16808)
print(f"Health score: {audit.health_score}%")
import { SynupClient } from 'synup-js'

const client = new SynupClient({ apiKey: "YOUR_API_KEY" })

// Fetch all locations
const locations = await client.locations.list({ first: 10 })

// Get reviews for a location
const reviews = await client.reviews.list({ locationId: 16808, first: 20 })

// Respond to a review
await client.reviews.respond("review-uuid", "Thank you for your feedback!")

Install in seconds

One command. No heavy dependencies.

Install the SDK

pip install synup-sdk

Settings → Integrations → Generate in your Synup workspace

3

Start building

client = synup.Synup()

Why use the SDK instead of the API?

Ship in hours, not weeks. The SDK handles the hard parts so you can focus on building.

⚡

Pre-built workflows

Auto-reply to reviews, bulk onboard locations, generate reports — one function call.

🔄

Auto-retry + rate limits

Built-in exponential backoff on 429/5xx. Your scripts survive API hiccups.

📄

Auto-pagination

Call .auto_paging_iter() and iterate all results. No cursor tracking.

📌

Typed responses

Access location.name instead of result["locations"][0]["name"]. IDE autocomplete works.

📖

Specific errors

Catch RateLimitError, AuthenticationError, NotFoundError — not generic 400s.

🛠

Zero boilerplate

Reads API key from env. Default timeouts. Sensible defaults. Two lines to start.
